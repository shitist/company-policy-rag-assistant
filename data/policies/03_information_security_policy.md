# Information Security Policy

Sample policy document for a fictional organization. This file contains no real company data.

## Purpose

This policy defines basic information security expectations for the sample organization.

## Confidential Information Handling

Staff must not place confidential information into public tools, unmanaged storage, personal accounts, or unapproved communication channels. Demo examples of confidential information include private schedules, unreleased creative material, personal data, credentials, and non-public incident details.

When a task requires an example, staff should use neutral placeholders rather than real names, files, or operational details.

## Account and Device Hygiene

Staff should use unique credentials, multifactor authentication where available, and managed devices for work-related access. Shared passwords, screenshots of access tokens, and credential storage in plain text are not allowed.

## Security Event Reporting

If staff suspect an account compromise, lost device, exposed credential, or unauthorized access, they should stop further sharing, preserve available evidence, and notify the fictional security review contact defined in the demo workflow. Public communication should not mention security details unless the incident response process says so.

## AI and External Tool Security

AI tools, public paste sites, browser extensions, and file conversion services should be treated as external tools unless explicitly approved. Staff should avoid uploading private or identifying content to those services.
